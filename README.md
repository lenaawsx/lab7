# Лабораторная работа №7. jQuery – сравнительный анализ

## Ответы на вопросы

### 1. Какие jQuery-функции использовались?
- `$(document).ready()` — ожидание загрузки DOM  
- `.click()` — обработчик событий  
- `.addClass()` / `.removeClass()` — переключение классов  
- `.data()` — работа с data-атрибутами  
- `.animate()` — плавная прокрутка  
- `$.get()` — AJAX-запрос  
- jQuery UI: `.draggable()`, `.datepicker()`  

### 2. Что показалось быстрее/проще — jQuery или чистый JS?
- jQuery проще и короче: меньше кода, удобные методы, легкая работа с DOM.  
- Vanilla JS немного длиннее, но стал удобным благодаря современным функциям (`classList`, `fetch`, `scrollIntoView`).  

### 3. Что из этого вы бы оставили, если писали современный сайт?
- Для современных проектов чаще достаточно чистого JavaScript — он теперь закрывает почти все задачи.  
- jQuery можно использовать только при необходимости старых плагинов или лёгкого прототипирования.  
- Для сложных UI-компонентов лучше применять современные библиотеки (Vue/React), а не jQuery UI.
